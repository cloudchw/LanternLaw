# lanternlaw 时间段管理功能测试计划

## 🎯 测试目标
测试"我的"页面中的时间段管理功能，包括添加、修改、删除时间段以及重叠检测。

## 🧪 测试环境
- **设备**: HarmonyOS 模拟器
- **应用**: lanternlaw (掌灯人)
- **测试框架**: HarmonyOS HiTest

## 📋 测试用例

### 1. 添加时间段测试 (TimeRangeTest.ets)
**C1-001**: 成功添加新时间段
- 前置条件：打开"我的"页面
- 测试步骤：
  1. 点击"+ 添加"按钮
  2. 验证时间配置窗口弹出
  3. 设置开始时间为 22:30
  4. 设置结束时间为 06:30
  5. 验证管控时长显示为 8.00 小时
  6. 输入标签名称"夜间休息"
  7. 点击"确定"按钮
- 预期结果：时间段卡片显示"夜间休息 22:30 → 06:30 (跨天)"

**C1-002**: 默认时间显示
- 测试步骤：点击"+ 添加"按钮
- 预期结果：开始时间和结束时间都显示为 00:00

**C1-003**: 标签名称为空
- 测试步骤：不输入标签名称直接点击"确定"
- 预期结果：显示默认标签名称"新时间段"

### 2. 修改时间段测试
**C2-001**: 修改现有时间段
- 前置条件：已存在"夜间休息"时间段
- 测试步骤：
  1. 点击"夜间休息"时间段卡片
  2. 修改开始时间为 23:00
  3. 修改结束时间为 07:00
  4. 点击"确定"按钮
- 预期结果：时间段更新为"23:00 → 07:00 (跨天)"

### 3. 时间重叠测试
**C3-001**: 时间段重叠检测
- 前置条件：已存在"夜间休息 22:30 → 06:30"
- 测试步骤：
  1. 尝试添加时间段"21:00 → 08:00"
  2. 点击"确定"按钮
- 预期结果：显示重叠警告，时间段未添加

**C3-002**: 无重叠时间段添加
- 测试步骤：
  1. 添加时间段"09:00 → 12:00"
  2. 点击"确定"按钮
- 预期结果：时间段成功添加

### 4. 跨天计算测试 (TimeRangeTest.ets)
**C4-001**: 跨天时间计算 (自动化测试)
- 测试用例：testCrossDayCalculation
- 测试数据：22:00 → 06:00
- 预期结果：管控时长 = 8.00 小时

**C4-002**: 同天时间计算
- 测试步骤：设置 09:00 → 12:00
- 预期结果：管控时长 = 3.00 小时

### 5. 删除时间段测试 (TimeRangeTest.ets)
**C5-001**: 删除时间段
- 前置条件：已存在多个时间段
- 测试步骤：点击时间段卡片右侧的"×"按钮
- 预期结果：时间段从列表中移除

**C5-002**: 删除最后时间段
- 前置条件：只有一个时间段
- 测试步骤：点击"×"按钮
- 预期结果：显示警告，不能删除（至少保留一个时间段）

### 6. 启用/禁用时间段测试
**C6-001**: 启用时间段
- 测试步骤：点击时间段卡片上的开关
- 预期结果：时间段启用，时间显示为金色

**C6-002**: 禁用时间段
- 测试步骤：再次点击时间段卡片上的开关
- 预期结果：时间段禁用，时间显示为灰色
