# 账号管理功能设计

## 概述

在"我的"Tab 中新增"账号管理"卡片，支持用户创建账号（用户名 + emoji头像 + 密码），以及修改用户名、修改头像、修改密码、找回密码功能。账号数据保存在本地 RDB 数据库中。

## 需求

- 首次使用 App 时强制创建账号，创建后才能进入主界面
- 支持设置用户名（2-12字符）、emoji 头像（从内置列表选择）、密码（6-20字符）
- 支持修改用户名、修改头像、修改密码
- 找回密码通过 HarmonyOS 系统锁屏验证，验证通过后可重置密码
- 密码以 SHA-256 哈希存储，不存明文

## 交互模式

采用弹窗模式，与现有白名单、时间管理卡片保持一致。所有操作通过全屏遮罩 + 居中弹窗完成，不跳转子页面。

## 卡片设计

### 位置

在"提醒设置"卡片之后、"关于"卡片之前（第五张卡片）。

### 布局（已创建账号）

```
┌─────────────────────────────────────┐
│  👤  账号管理                        │
│─────────────────────────────────────│
│  修改用户名                     >   │
│─────────────────────────────────────│
│  修改头像                       >   │
│─────────────────────────────────────│
│  修改密码                       >   │
│─────────────────────────────────────│
│  找回密码                       >   │
└─────────────────────────────────────┘
```

- 卡片样式与现有卡片一致：`COLORS.card` 背景、`COLORS.border` 边框、`borderRadius(12)`、`padding(16)`
- 头像 emoji 居中显示，fontSize(48)
- 用户名居中显示在头像下方，fontSize(16)，fontColor `COLORS.text`
- 操作列表每项用分割线隔开，右侧显示 `>` 箭头指示

## 弹窗设计

所有弹窗复用现有全屏遮罩模式：`Column` 全屏 + `rgba(0,0,0,0.6)` 背景 + 居中卡片。

### 创建账号弹窗

- 宽度 88%，圆角 20，与 `TimePickerDialog` 一致
- 标题："创建你的账号"
- 内容从上到下：头像选择区域（点击跳转头像选择弹窗）→ 用户名输入 → 密码输入 → 确认密码输入 → 取消/创建按钮
- 密码输入使用 `InputType.Password`
- 验证失败时在输入框下方显示红色提示

### 选择头像弹窗

- 4行6列网格，24个预设 emoji
- emoji 列表：😊😄😎🤓😇🥳 🦊🐱🐶🐼🦁🐯 🌙⭐🔥💪🎯🏆 🌸🍀🎨📚🧘💡
- 选中项用金色边框高亮（`COLORS.gold`）
- 底部取消/确认按钮

### 修改用户名弹窗

- 标题："修改用户名"
- 一个 TextInput，预填当前用户名
- 取消/保存按钮

### 修改密码弹窗

- 标题："修改密码"
- 三个 TextInput（从上到下）：旧密码、新密码、确认新密码
- 均使用 `InputType.Password`
- 取消/保存按钮

### 找回密码弹窗

- 标题："找回密码"
- 初始状态：提示文字 + "验证系统锁屏密码"按钮
- 点击按钮调用 `@ohos.userAuthentication` 系统身份验证
- 验证通过后：显示新密码和确认密码输入框
- 取消/重置按钮

## 技术方案

### 数据存储

在 RDB `lanternlaw_config` 表中以 key `'user_account'` 存储账号信息 JSON：

```json
{
  "username": "掌灯人",
  "avatar": "😊",
  "passwordHash": "sha256hashedvalue",
  "createdAt": 1712000000000,
  "updatedAt": 1712000000000
}
```

### 密码哈希

使用 `@ohos.security.crypto` 的 SHA-256 对密码进行哈希，加盐处理。不存储明文密码。

### 系统锁屏验证

使用 `@ohos.userAuthentication.getUserAuthInstance()`：
- 挑战类型：`ALL`（PIN、指纹、人脸等）
- 信任级别：`CROSS_DEVICE`
- 验证成功后解锁密码重置界面

### 账号状态检测

App 启动时读取 `user_account` key：
- 不存在 → 全屏展示创建账号弹窗，阻止进入主界面
- 已存在 → 正常进入主界面，卡片展示账号信息

## 验证规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 用户名 | 必填，2-12个字符 | "用户名需要2-12个字符" |
| 密码 | 必填，6-20个字符 | "密码需要6-20个字符" |
| 确认密码 | 必须与密码一致 | "两次输入的密码不一致" |
| 旧密码（修改时） | 必须与存储的哈希匹配 | "旧密码不正确" |

所有验证失败在对应输入框下方显示红色提示文字（`#ff4444`，fontSize(12)）。
