# 掌灯人 - 多帐号家庭体系 产品设计文档

> 版本: v1.0 | 日期: 2026-04-07 | 状态: 待评审

---

## 一、背景与目标

### 1.1 现状
掌灯人目前是单帐号应用，所有配置（灯律时段、白名单应用）全局生效，仅存储一个 `UserAccount` 和一份 `LanternLawConfig`。

### 1.2 目标
构建多帐号家庭体系，支持家长与儿童在不同设备上使用，家长可远程为儿童配置管控规则，并通过消息通知进行沟通激励。

### 1.3 核心价值
- **家长**：远程管理孩子的屏幕使用时间，不必物理接触孩子设备
- **儿童**：在家长的引导下养成自律习惯，通过消息获得鼓励和提醒
- **家庭**：形成数字化家校沟通的自律管理闭环

## 二、帐号模式

### 2.1 两种模式

| 维度 | 家长模式 | 儿童模式 |
|------|---------|---------|
| 标识 | 👨‍👩‍👧‍👦 家长 | 👦/👧 儿童 |
| 管控设置 | 完全控制（自己的 + 孩子的） | 只读查看（由家长配置） |
| 沟通组 | 创建和管理沟通组 | 加入家长的沟通组 |
| 消息 | 发送自定义消息、快捷消息 | 发送快捷回复、查看家长消息 |
| 数据统计 | 查看自己 + 孩子的数据 | 仅查看自己的数据 |
| 帐号管理 | 修改用户名/头像/密码 | 修改用户名/头像/密码 |

### 2.2 帐号创建流程

```
创建帐号（用户名 + 头像 + 密码）
        │
        ▼
  ┌─────────────┐
  │  选择身份    │
  │             │
  │ 👨‍👩‍👧‍👦 家长   👦 儿童 │
  └─────┬───────┘
        │
   ┌────┴────┐
   │         │
   ▼         ▼
家长路径    儿童路径
```

**家长路径**：选择家长模式 → 创建沟通组（输入组名）→ 进入主页

**儿童路径**：选择儿童模式 → 输入家长的邀请码 → 等待家长确认 → 进入主页

### 2.3 帐号与设备关系

- **一台设备一个帐号**，帐号创建后绑定到当前设备
- 家长和孩子分别在自己的设备上安装和使用掌灯人
- 每个帐号有唯一的 `accountId`（UUID），作为跨设备识别标识

## 三、沟通组（家庭组）

### 3.1 沟通组模型

```
沟通组「李家」
├── 👨 爸爸（家长）- 管理员
├── 👦 小明（儿童）- 绑定到爸爸
└── 👧 小红（儿童）- 绑定到爸爸
```

- 一个沟通组由 **1个家长** + **多个儿童** 组成
- 家长是沟通组的创建者和管理员
- 儿童帐号必须绑定到一个家长帐号
- 沟通组名称由家长设定（如"李家"、"张家"）

### 3.2 邀请加入流程

#### 家长生成邀请码
1. 家长进入 ProfileTab → 沟通组管理卡片
2. 点击「邀请孩子加入」
3. 系统生成 6 位邀请码（如 `A3X9K2`），有效期 24 小时
4. 邀请码可复制到剪贴板，或通过系统分享面板发送
5. 每个邀请码仅限 1 个儿童使用，使用后自动失效

#### 儿童加入沟通组
1. 儿童在创建帐号选择"儿童模式"后，进入邀请码输入界面
2. 输入家长提供的 6 位邀请码
3. 系统验证邀请码有效性
4. 验证通过后，儿童端显示「等待家长确认...」
5. 家长端收到通知：「小明 请求加入沟通组」
6. 家长点击「确认」后，绑定关系建立
7. 家长端收到「小明已加入沟通组」通知
8. 儿童端收到「已加入沟通组」通知，进入主页

#### 安全机制
- 邀请码 6 位字母数字，36^6 ≈ 22 亿组合
- 每分钟最多 5 次尝试，防止暴力破解
- 24 小时过期
- 使用一次后失效，需重新生成
- 家长可在 30 分钟内拒绝加入请求
- 儿童无法主动退出沟通组，只有家长可移除儿童

## 四、管控配置体系

### 4.1 配置归属

| 配置项 | 谁配置 | 谁生效 | 存储位置 |
|--------|--------|--------|---------|
| 家长自己的灯律时段 | 家长 | 家长设备 | 家长设备本地 + 云端 |
| 家长自己的白名单 | 家长 | 家长设备 | 家长设备本地 + 云端 |
| 儿童的灯律时段 | 家长（远程） | 儿童设备 | 云端 → 同步到儿童设备 |
| 儿童的白名单 | 家长（远程） | 儿童设备 | 云端 → 同步到儿童设备 |
| 儿童的全局开关 | 家长（远程） | 儿童设备 | 云端 → 同步到儿童设备 |

### 4.2 家长配置儿童的管控规则

家长在 ProfileTab 中进入「孩子管理」面板，选择某个儿童后，可以为该儿童配置：

1. **启用/停用掌灯人** — 远程开关儿童的灯律管控
2. **灯律时段** — 设置儿童的灯律时间范围（可添加多个时段）
3. **白名单应用** — 选择儿童在管控时段内可使用的应用

配置保存后自动同步到儿童的设备。

### 4.3 配置同步机制

```
家长设备                      云端                       儿童设备
    │                          │                           │
    │── 保存儿童配置 ──────────>│                           │
    │                          │── 推送通知 ──────────────>│
    │                          │                           │── 拉取最新配置
    │                          │                           │── 更新本地RDB
    │                          │                           │── 应用ScreenTimeGuard策略
    │                          │                           │── 显示通知「家长更新了灯律设置」
    │                          │                           │
    │<── 拉取儿童自律数据 ─────│<── 上传每日记录 ──────────│
```

**同步策略**：
- **实时同步**：配置变更后通过推送通知立即触发儿童端拉取
- **定时兜底**：每 5 分钟检查一次配置版本更新
- **启动同步**：应用每次回到前台时自动同步
- **离线处理**：儿童设备离线时，云端保存最新配置；上线后自动同步

## 五、消息通知系统

### 5.1 消息类型

| 类型 | 方向 | 说明 |
|------|------|------|
| 🔔 提醒 | 家长→儿童 | 「该休息了」「注意保护眼睛」 |
| 💪 鼓励 | 家长→儿童 | 「今天表现很棒」「继续保持」 |
| 💬 自定义 | 家长→儿童 | 家长自由输入的消息 |
| 🙋 请求 | 儿童→家长 | 「我想多玩10分钟」 |
| ✅ 反馈 | 儿童→家长 | 「我完成了今天的作业」 |
| ⚙️ 系统 | 双向 | 配置更新通知、加入/退出沟通组通知 |

### 5.2 快捷消息模板

**家长发送给儿童：**
- 「该休息了，早点睡觉 🌙」
- 「今天表现很棒，继续保持 💪」
- 「专注完成作业，加油 📚」
- 「注意保护眼睛，看看远处 👀」
- 「记得喝水 💧」

**儿童发送给家长：**
- 「我完成了今天的作业 ✅」
- 「我想多玩10分钟可以吗？ 🙏」
- 「我今天表现很好 🌟」

### 5.3 消息交互流程

#### 家长发消息
1. 在孩子管理面板或消息页面，选择一个儿童
2. 从快捷模板中选择，或输入自定义消息
3. 消息通过云端推送到儿童设备

#### 儿童接收消息
- **灯律管控时段内**：消息以常驻通知形式展示（不可划走），儿童需点击「我知道了」确认
- **灯律管控时段外**：普通通知栏通知 + 应用内消息卡片
- 儿童可从快捷模板中选择回复

#### 消息历史
- 双方均可查看完整消息历史记录
- 消息按时间倒序排列，区分发送/接收方向

## 七、数据架构

### 7.1 新增数据模型

```
UserAccount（扩展）
├── accountId: string (UUID, 新增)
├── username: string
├── avatar: string
├── passwordHash: string
├── accountMode: 'parent' | 'child' (新增)
├── familyGroupId: string (新增)
├── parentAccountId: string (新增, 儿童专用)
├── createdAt: number
└── updatedAt: number

FamilyGroup（新增）
├── groupId: string (UUID)
├── groupName: string
├── parentAccountId: string
├── invitationCode: string
├── invitationExpiresAt: number
├── createdAt: number
└── updatedAt: number

Message（新增）
├── id: string (UUID)
├── senderAccountId: string
├── receiverAccountId: string
├── messageType: 'encouragement' | 'reminder' | 'custom' | 'request' | 'feedback' | 'system'
├── content: string
├── isRead: boolean
└── createdAt: number
```

### 7.2 存储策略

| 数据 | 本地存储 | 云端存储 | 同步方向 |
|------|---------|---------|---------|
| 帐号信息 | RDB accounts 表 | Cloud DB | 双向 |
| 沟通组 | RDB family_groups 表 | Cloud DB | 双向 |
| 灯律配置 | RDB key-value (按 accountId) | Cloud DB | 家长→儿童 |
| 每日记录 | RDB daily_records 表 | Cloud DB | 儿童→家长 |
| 消息记录 | RDB messages 表 | Cloud DB | 双向 |
| 设备绑定 | Preferences | 不同步 | 仅本地 |

### 7.3 配置隔离

当前配置 key `'lanternlaw_config'` 改为按帐号隔离：
- `'lanternlaw_config_{accountId}'` — 每个帐号独立的灯律配置
- 儿童的配置由家长写入云端，儿童设备从云端拉取后存入本地

## 八、技术选型

| 能力 | 技术方案 | 说明 |
|------|---------|------|
| 跨设备数据同步 | HarmonyOS Cloud DB (`@kit.CloudKit`) | 支持离线缓存、冲突解决、实时推送 |
| 实时消息推送 | Push Kit (`@kit.PushKit`) | 配合 Cloud DB 触发器实现即时通知 |
| 本地数据存储 | RDB（扩展现有数据库） | 新增 accounts、family_groups、messages 表 |
| 邀请码生成 | UUID + 随机字符截取 | 简单可靠 |
| 消息通知展示 | `@ohos.notificationManager` | 灯律时段内使用常驻通知 |

## 九、实施分阶段建议

### Phase 1: 本地多帐号 + 家长儿童模式（4-6 周）
- 扩展 UserAccount 模型，新增 accounts/family_groups/messages 表
- 帐号创建后选择模式（家长/儿童）
- 儿童模式 ProfileTab 只读化
- 沟通组创建和邀请码（本地验证）
- 家长为儿童配置灯律和白名单（本地存储）
- 基础消息功能（本地模拟）
- 现有单帐号用户迁移

### Phase 2: 云端同步 + 跨设备（8-12 周）
- Cloud DB 接入，帐号/沟通组/配置/消息云端存储
- Push Kit 接入，配置变更和消息实时推送
- 家长远程配置同步到儿童设备
- 儿童自律数据同步给家长查看
- 二维码邀请（替代手动输入邀请码）

### Phase 3: 增强功能（后续）
- 每日屏幕时间预算（不仅限于时段管控）
- 多家长（爸爸+妈妈）共同管理
- 周报自动发送给家长
- 成就分享

## 十、现有用户迁移

### 10.1 数据库迁移（v2 → v3）
1. 新增 accounts、family_groups、messages 表
2. 将现有 user_account 迁移到 accounts 表，生成 accountId
3. 将现有 lanternlaw_config key 改名为 lanternlaw_config_{accountId}
4. 数据库版本升级到 3

### 10.2 模式选择引导
升级后首次启动，检测帐号无 accountMode，弹出全屏引导：
- 「欢迎升级掌灯人！请选择你的身份」
- 必须选择后才能继续使用（不可跳过）
- 选择家长 → 引导创建沟通组
- 选择儿童 → 引导输入邀请码（可稍后）

## 十一、关键设计原则

1. **家长权威**：儿童的所有管控设置由家长决定，儿童无法修改
2. **数据隔离**：每个帐号的配置和数据独立存储，互不干扰
3. **离线可用**：儿童设备离线时仍按最新配置执行管控，上线后自动同步
4. **安全第一**：邀请码防暴力破解、设备绑定防帐号盗用、配置变更需密码验证
5. **渐进体验**：Phase 1 先跑通本地多帐号流程，Phase 2 再上云端跨设备
