2. 无法选择或修改起始时间、结束时间
3. 新建的时间管控规则无法创建

## 🔍 根本原因分析

### 逻辑冲突
在之前的修复中，我们存在一个**逻辑冲突**：

**1. 全天管控计算修复** (ProfileTab.ets:84-98行)
```typescript
// 在 calculateAddingDuration() 中添加了全天管控支持
if (startMinutes === endMinutes) {
  return '24.00h'; // 全天管控
}
```

**2. 时间段添加验证** (ProfileTab.ets:242-245行) ❌ **冲突代码**
```typescript
if (startMinutes === endMinutes) {
  Logger.warn('ProfileTab', '开始时间和结束时间不能相同');
  return; // 这里阻止了全天管控的添加！
}
```

### 冲突说明
- **计算逻辑**允许全天管控 (00:00 → 00:00 = 24小时)
- **验证逻辑**拒绝全天管控 (开始时间不能等于结束时间)
- **结果**: 用户点击确认按钮时，验证失败直接返回，没有反应

## ✅ 修复方案

### 移除冲突的验证逻辑
```typescript
// 删除了这段验证代码
- if (startMinutes === endMinutes) {
-   Logger.warn('ProfileTab', '开始时间和结束时间不能相同');
-   return;
- }
```

### 修复后的逻辑
```typescript
// 只检查重叠，允许全天管控
const overlap = TimeRangeManager.checkOverlap(newRange, this.timeRanges);
if (overlap) {
  Logger.warn('ProfileTab', '时间段与现有时间段重叠');
  return;
}
```
